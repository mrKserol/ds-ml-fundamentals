# Models

В этом разделе собраны реализации базовых и ансамблевых алгоритмов
машинного обучения для задачи регрессии.

Основной фокус — **понимание алгоритмов “изнутри”**:
как они строятся, обучаются и почему работают,
а также сравнение собственных реализаций с эталонными из `sklearn`.

---

## Структура каталога

### [decision_tree/](decision_tree)

Реализация дерева решений для регрессии (`DecisionTreeRegressor`) с нуля.

**Ключевые особенности:**
- тип задачи: регрессия
- критерий разбиения: Mean Squared Error (MSE)
- стратегия построения: depth-wise (в глубину)
- предсказание в листе: среднее значение таргета
- формат предсказаний: `float` (совместимо со `sklearn`)

В ноутбуке подробно разобраны:
- критерий MSE и взвешенный MSE,
- поиск лучшего разбиения,
- рекурсивное построение дерева,
- сравнение с `sklearn.tree.DecisionTreeRegressor`,
- визуальный анализ предсказаний и ошибок.

---

### [gradient_boosting/](gradient_boosting)

Реализация градиентного бустинга для регрессии с ручной логикой обновления ансамбля.

**Ключевые особенности:**
- тип задачи: регрессия
- функция потерь: Mean Squared Error (MSE)
- базовый алгоритм: `DecisionTreeRegressor`
- обучение на антиградиентах функции потерь
- пошаговое обновление предсказаний с `learning_rate`

В ноутбуке показаны:
- вывод градиента для MSE,
- обучение ансамбля по остаткам,
- сравнение с `sklearn.GradientBoostingRegressor`,
- анализ метрик качества и визуализация ошибок.

---

## Проверка корректности реализаций

Для всех моделей:
- используется одинаковое разбиение данных на train/test,
- сравниваются метрики качества (MSE, MAE),
- проводится визуальный анализ предсказаний.

Небольшие численные расхождения с `sklearn` считаются допустимыми
и объясняются различиями в выборе порогов, порядке вычислений
и численных эффектов.

---

## Цель раздела

Каталог `models` предназначен для:
- углублённого понимания классических ML-алгоритмов,
- демонстрации инженерного подхода к реализации моделей,
- подготовки к работе с ансамблевыми методами
  (Random Forest, Gradient Boosting, XGBoost и др.).

Реализации ориентированы на учебные и исследовательские цели,
а не на промышленное использование.
