# Метрики в машинном обучении

В этом разделе собраны основные метрики качества,
используемые в задачах машинного обучения.

Метрики сгруппированы по типу задачи:
- регрессия,
- классификация,
- ранжирование (поиск и рекомендации).

Основной фокус сделан на понимании смысла метрик,
их ограничений и корректной интерпретации результатов,
а не только на использовании готовых библиотек.

## Структура раздела

### Регрессия
**Папка:** [regression/](regression)

Метрики для задач прогнозирования числовых величин.

Рассматриваются:
- MAE, MSE, RMSE
- R²
- MAPE и sMAPE

Метрики реализуются вручную и сравниваются с реализациями из `sklearn`
для валидации корректности.

### Классификация
**Папка:** [classification/](classification)

Метрики для бинарной классификации,
основанные на матрице ошибок (confusion matrix).

Рассматриваются:
- Accuracy
- Precision
- Recall
- F1-score

Отдельное внимание уделено влиянию дисбаланса классов
и компромиссам между метриками.

### Ранжирование
**Папка:** [ranking/](ranking)

Метрики для задач поиска и рекомендаций,
где модель формирует упорядоченный список объектов,
а не бинарное решение.

Рассматриваются:
- Recall@K
- Precision@K
- F1@K
- Specificity@K

Метрики оценивают качество выдачи в верхней части списка (top-K),
что соответствует реальному пользовательскому поведению.

## Общие принципы

- Метрики реализуются вручную для понимания формул и логики расчёта.
- Использование библиотек (например, `sklearn`) применяется
  только для проверки корректности.
- Визуализации используются там, где они помогают
  лучше понять поведение метрик.
- Выбор метрики всегда зависит от постановки задачи
  и продуктовых требований.
